## Coding conventions

- **Split files logically** — prefer many small, focused files over large ones.
- **Docs and comments in English** — all code comments, JSDoc, and inline documentation must be written in English.
- **Comment the why, not the what** — comment non-trivial logic, edge cases, and reasons behind decisions. Skip comments for self-explanatory code.
- **No statement cramming on one line** — never join multiple statements with semicolons on a single line. Each statement goes on its own line.

## Git

- Never add a Claude co-author signature to commits.
